# SEQUENTIAL THINKING RULE

## Mục tiêu

Dùng Sequential Thinking để phân tích task phức tạp trước khi thực thi, không dùng cho mọi tình huống.

## Bắt buộc dùng khi

- debug phức tạp
- kiến trúc / thiết kế
- brainstorm / so sánh phương án
- refactor nhiều file
- deploy có rủi ro
- task nhiều bước hoặc nhiều ràng buộc

## Không bắt buộc dùng khi

- chào hỏi / xã giao
- câu hỏi fact đơn giản
- sửa nhỏ, hiển nhiên
- format / wording đơn giản
- checklist cố định

## Thứ tự chuẩn

1. Thu thập context tối thiểu
2. Dùng Sequential Thinking để phân tích
3. Chọn workflow / skill phù hợp
4. Thực thi
5. Tự kiểm tra nếu cần

## Cấu trúc mỗi bước

- Objective
- Evidence
- Hypothesis / Decision
- Next Action
- Continue?

## Điều kiện dừng

- đủ thông tin để hành động
- đã chọn được hướng rõ ràng
- không có insight mới sau 2 bước
- task nhỏ hơn dự kiến

## Ràng buộc

- Không overthink
- Không lặp ý
- Không dùng Sequential để thay workflow
- Không mở quá nhiều nhánh nếu chưa có bằng chứng
